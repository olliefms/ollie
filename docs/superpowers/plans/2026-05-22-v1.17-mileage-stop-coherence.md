# v1.17 ŌĆö Mileage & Stop Display Coherence

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to implement task-by-task. Each task includes its TDD steps. Do not edit this plan file from inside an implementer agent ŌĆö escalate to the orchestrator instead.

**Goal:** Make trip mileage data agent-maintainable (recalc, repair, MCP coverage) and apply one coherent stops-table model across dispatcher load detail, dispatcher trip detail, driver trip detail, and driver stop detail. Normalize stop timezone handling and add a migration safety net.

**Scope:** Closes #259, #260, #133, #251, #253, #254, #255, #256, #257, #258, #239. Closes #212 as superseded.

**Architecture:** Three coupled design decisions drive most of the surface area:

1. **One stops-table shape, two contexts.** Loads render stops with a per-leg `miles` column and `Total Miles` footer. Trips render the same table, prefixed with a synthetic `Stop 0 (Origin)` row carrying the previous trip's last facility (when `mileage_summary.origin` is present). No more standalone "Mileage card" anywhere.
2. **`actual_arrive` / `actual_depart` are naive-local + UTC companion.** Stop times stored as naive strings with `timezone` carrying tz; API returns `*_utc` companions computed from naive + tz. New writes with a `Z` or offset suffix get 422. Existing data stays readable.
3. **Detention is computed, not stored.** Driver stop detail derives early/on-time/late, dwell, and detention live from `actual_arrive`, `actual_depart`, `scheduled_arrive`, `scheduled_arrive_end`, and a server config `OLLIE_FREE_DWELL_MINUTES` (default 120).

**Tech Stack:** Rust (axum, serde), Arrow/LanceDB storage, vanilla JS frontends (`static/dispatch/app.js`, `static/driver/pages/*.js`).
