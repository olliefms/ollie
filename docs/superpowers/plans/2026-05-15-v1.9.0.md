# v1.9.0 Implementation Plan

**Goal:** Fix the critical v1.8.0 trip schema migration bug (#135), three driver UI bugs (#137, #138, #139), add SPA navigation history stack (#141), dispatch home dashboard (#143), and newest-first sort on all list endpoints (#136).

**Architecture:** #135 is a DB-layer hotfix following the `open_or_create_facility` pattern. UI bugs are isolated CSS/JS changes. #141 is a pure-frontend refactor enabling the #143 home dashboard. #136 adds in-memory sorts after LanceDB query results are collected (same pattern as `list_trips`). KPI count endpoints (#143) are thin handlers over existing `count_rows` calls.

**Tech Stack:** Rust, Axum 0.7, LanceDB 0.27, Arrow 57, vanilla JS SPA
