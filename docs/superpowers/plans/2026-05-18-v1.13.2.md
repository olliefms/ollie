## Task 2: Driver SPA — fetch version, drop hardcoded constant (issue #203, client half)

- [ ] **Step 1: Create the version utility**

Create `static/driver/utils/version.js`:

```js
let cached = null;

export async function getAppVersion() {
  if (cached !== null) return cached;
  try {
    const r = await fetch('/version');
    if (!r.ok) return cached = '—';
    const data = await r.json();
    return cached = (data && typeof data.version === 'string') ? `v${data.version}` : '—';
  } catch (_) {
    return cached = '—';
  }
}
```

The cache is module-scoped so subsequent Account renders in the same session don't re-fetch.

## Task 4: Trip-detail duplicate render investigation (issue #198)

**Files:**
- Modify: `static/driver/pages/trip-detail.js` (likely)

- [ ] **Step 1: Reproduce locally**

If you cannot reproduce, dispatch a subagent to deploy a Playwright-driven repro (the screenshots in the issue are authoritative — believe the issue if you cannot reproduce locally).

- [ ] **Step 2: Diagnose**

Likely root causes to check, in order:

1. **Double-invocation of `route()`** — `app.js` calls `route()` once on module load (line 98) and once per `popstate`. If a `navigate(...)` from elsewhere fires during initial render, two `renderTripDetail` calls may race. Check whether `container.replaceChildren()` (line 15) is hit only once by adding a `console.count('trip-detail render')` temporarily and reloading.
2. **`apiFetch('/me')` race with render** — line 19 awaits `/me` after `replaceChildren()` but before the rest of the DOM is built. If a second route invocation fires while this `await` is in flight, the second render will append to a container that has already been cleared — producing duplicate output.
3. **Lingering DOM from a previous view** — confirm `renderTrips` and other sibling pages also clear `app` before appending.

- [ ] **Step 3: Apply the minimal fix**

If the cause is (1) or (2), the minimal fix is to mark the render as "in flight" and ignore re-entry:

In `static/driver/pages/trip-detail.js`, near the top of `renderTripDetail` (just after `if (!isAuthenticated())` returns), insert:

```js
const renderToken = Symbol('trip-detail-render');
container.__renderToken = renderToken;
```

Then at every place the function appends to `page` (or to `container` indirectly), guard with:

```js
if (container.__renderToken !== renderToken) return;
```

The simplest placement: just after `const data = await apiFetch(...)` and just after each subsequent `await` (e.g. `await refreshDocs()`). If a newer render has started, the older one bails out before producing duplicate DOM.

If the cause is (3) instead, fix it at the appropriate sibling page by adding `container.replaceChildren()` at the start of its render function.

## Task 8: Driver doc preview returns Unauthorized (issue #202)

**Files:**
- Modify: `static/driver/pages/trip-detail.js`
- Modify: `static/driver/css/components.css`

**Background:** The preview overlay sets `iframe.src = '/driver/api/v1/trips/.../content'` directly. An iframe `src=` request cannot carry the `Authorization: Bearer` header, so the server's auth middleware rejects it with 401. The server endpoint already exists and accepts the driver JWT — the bug is entirely client-side.

**Fix:** `fetch()` the content with the Bearer header, build a blob URL via `URL.createObjectURL`, and assign that to the iframe (for PDFs) or to an `<img>` (for images). Revoke the blob URL on close to avoid leaks. Driver uploads are mostly JPG (camera) so this avoids the Brave-on-blob-PDF complication seen in #184; for the rare PDF case, the dispatcher's #184 caveat applies but is acceptable for the driver portal.

- [ ] **Step 1: Replace `openDocPreview` in trip-detail.js**

Replace with:

```js
async function openDocPreview(doc) {
  const overlay = document.createElement('div');
  overlay.className = 'doc-preview';

  const close = document.createElement('button');
  close.type = 'button';
  close.className = 'doc-preview__close';
  close.textContent = '×';
  close.setAttribute('aria-label', 'Close preview');

  let blobUrl = null;
  const teardown = () => {
    if (blobUrl) URL.revokeObjectURL(blobUrl);
    if (overlay.parentNode) overlay.parentNode.removeChild(overlay);
  };
  close.addEventListener('click', teardown);

  const spinner = document.createElement('div');
  spinner.className = 'doc-preview__spinner';

  overlay.appendChild(close);
  overlay.appendChild(spinner);
  document.body.appendChild(overlay);

  try {
    const r = await fetch(
      `/driver/api/v1/trips/${tripId}/documents/${doc.id}/content`,
      { headers: { 'Authorization': `Bearer ${getToken()}` } }
    );
    if (!r.ok) throw new Error(`Preview failed (${r.status})`);
    const blob = await r.blob();
    blobUrl = URL.createObjectURL(blob);

    spinner.remove();

    if (doc.mime_type === 'application/pdf') {
      const frame = document.createElement('iframe');
      frame.sandbox = '';
      frame.src = blobUrl;
      overlay.appendChild(frame);
    } else {
      const img = document.createElement('img');
      img.className = 'doc-preview__img';
      img.alt = doc.name;
      img.src = blobUrl;
      overlay.appendChild(img);
    }
  } catch (err) {
    spinner.remove();
    const errEl = document.createElement('div');
    errEl.className = 'doc-preview__error';
    errEl.textContent = err.message || 'Failed to load document';
    overlay.appendChild(errEl);
  }
}
```

Append 3–5 lessons to AGENTS.md following the existing rule + Why + How-to-apply structure used in the file. Lessons must reflect what was actually learned this sprint. Candidates (the orchestrator should triage the final list at PR time, not now — leave a placeholder anchor):

```markdown
- **Hardcoded version strings in the SPA drift on every release — fetch from a server endpoint instead.** v1.13.1 shipped with the Account page still showing v1.13.0 because `static/driver/pages/account.js` carried a hand-maintained `APP_VERSION` constant disconnected from `Cargo.toml`. v1.13.2 added `GET /version` and a small `utils/version.js` helper to fetch it once per session. — Why: every patch release will silently drift unless the version is sourced from `CARGO_PKG_VERSION` at runtime. — How to apply: when adding any new "what version am I running" UI affordance, build it on `/version` (or extend `/version` if more build metadata is needed). Never reintroduce a hardcoded version constant.
- **Iframe `src=` cannot carry an Authorization header — `fetch` + `URL.createObjectURL` is the JWT-friendly preview pattern.** The driver doc preview always returned 401 because the iframe could not authenticate. Fetch the content with the Bearer header, build a blob URL, and assign it to the `<iframe>` (PDFs) or `<img>` (images). Revoke the blob URL on close to avoid leaks. — Why: this is the same fundamental constraint that defeated the dispatcher preview in #184; the workaround is acceptable for images (most driver uploads) but still hits Brave's blob-PDF bug, so the dispatcher fix per #184 remains separate. — How to apply: any JWT-protected binary endpoint that needs in-page preview must fetch with the header and convert to a blob URL.
- **`app-bar.appendChild(badge)` after passing `right: backBtn` ends up between title and right slot.** With `.app-bar { justify-content: space-between }`, appending a third sibling to the bar produces a three-way space distribution: title left, badge centered, right slot right (or any variant of that). Always insert additional elements into the `.app-bar__right` slot — never directly on the bar. — Why: header layouts that mix flex children and slotted children look ad-hoc and break on small screens. — How to apply: when adding a badge or any control to an app-bar, target `appBar.querySelector('.app-bar__right')` and insert there.
- **Pages with a fixed bottom nav need `padding-bottom: var(--bottom-nav-height)` on the page container.** Defining the bottom-nav height as a token (`--bottom-nav-height` in `base.css`) lets every detail-view container reuse it consistently — and lets the bottom-nav rule itself reference the same token, eliminating a `calc()` duplication. — Why: any page with `min-height: 100vh` and no padding will have its last section clipped by the fixed nav. — How to apply: whenever adding a new page-level container that should scroll independently and sits under the bottom nav, add `padding-bottom: var(--bottom-nav-height)`.
- **Driver document filenames must be displayed as a human label, not the raw upload name.** Camera-captured filenames are 30+ random digits and look like noise. Display "DOCTYPE — Mon DD, HH:MM AM/PM" derived from `doc.tags['doctype:']` and `doc.created_at`; keep the original filename as a `title=` tooltip. Always CSS-truncate the title with `overflow:hidden;text-overflow:ellipsis;white-space:nowrap` as defense in depth. — Why: future doc sources (scanner uploads, email attachments) may also produce ugly names; the doctype + capture time is always meaningful. — How to apply: any list of user-uploaded files in the SPA should render a derived label, not `doc.name`.
```
